1. 회원 관리
    1.1 모든 회원 정보는 필수정보만 간결하게 입력받는다.
        - 선생님 : 이름, 아이디, 폰번호, 비밀번호
        - 학생 :  아이디, 비밀번호, 이름, 학교, 학년, 부모님 폰번호
        - 학부모 : 아이디, 비밀번호, 폰번호
    1.2 제휴 로그인 : 카카오, 네이버, 구글, 애플
    1.3 선생님 가입 시, 폰 번호와 인증메시지를 통해 가입인증을 처리한다.
    1.4 아이디 찾기 : 가입 시, 입력한 폰 번호로 인증 번호를 발송하고, 인증번호 입력후, 아이디를 찾을 수 있다.
    1.5 비밀번호 찾기 : 가입 시, 입력한 폰 번호로 인증 번호를 발송하고, 인증번호 입력후, 새로운 비밀번호를 입력할 수 있다.
    1.6 로그인 후 이동되는 메인 페이지
        - 선생님 : 메인 대시보드
        - 학생 : 마이페이지 → 수업코드 추가 후, 선생님이 가입을 승인하면 메인 대시보드가 활성화된다.
        - 학부모 : 메인 대시보드


2. 학습 관리
    2.1 teacher
        2.1.1 수업 관리 (전체)
            - 선생님은 자신이 수업 하는 과목을 분류해서 관리할 수 있다.
            (1) 일정 관리
                - 선생님은 수업 커리큘럼을 등록할 수 있다.
                - 과외 회차를 한 눈에 보고 편집할 수 있다.
                - 대시보드 메인화면에 바로 확인할 수 있도록 디스플레이 해 놓는다.
            (2) 교재 관리
                - 선생님은 ISBN으로 교재를 등록할 수 있고, 학생들이 최저가로 검색해서 살 수 있게 연결해준다.
            (3) 진도 관리
                - 선생님은 수업 진도를 기록하여 학생과 학부모가 진도 상황을 확인할 수 있게 한다.
            (4) 수업 자료
                - 클라우드를 활용해 수업에 필요한 문서나 영상을 업로드 가능하다.
        2.1.2 학생 관리 (학생 개개인의 수업)
            (1) 과제 관리
                - 선생님은 어떤 과제가 있는지 등록해 학생들이 확인할 수 있도록 한다.
            (2) 성적 관리
                - 선생님은 학생들의 성적을 한 눈에 확인할 수 있게 그래프로 만들어 기록
            (3) 출결 관리
                - 선생님은 수업이 시작되면 결석인 학생을 체크한다.
            (4) 학생 특징
                - 학생의 특성을 등록하고 조회할 수 있다.
                - 학생의 불량한 태도를 등록하고, 3회 등록됐을 시 학부모에게 알림이 간다.
            (5) 상담
                - 학부모와 상담한 내용을 기록한다.
        2.1.3 마이페이지
            (1) 수업 추가
                - 수업 추가 시, 고유 수업코드가 생성된다.
                - 고유 수업코드를 학생에게 전달시, 학생이 코드를 통해 가입한다.
                - 해당 코드를 통해 등록된 학생을 가입 승인, 거부할 수 있다.
                - 수업 시작일
                    - 선생님은 자신이 과외를 시작한 날짜를 등록해 놓는다.
                - 학생 정보 입력
        2.1.4 기타
            - 학생의 이름으로 검색 가능하게 한다.
            - 학생 이름 검색창 옆에 드롭다운 버튼으로 학생을 선택할 수 있도록 한다.
            - 강의 시작 시간이 되면, 저절로 해당 시간에 수업을 듣는 학생의 탭으로 전환된다.
    2.2 student
        (1) 과제 관리
            - 학생의 대시보드 메인에 ‘오늘의 과제’가 디스플레이 해서 선생님이 올린 과제를 쉽게 확인할 수 있도록 한다.
        (2) 성적 관리
            - 학생은 자신의 학교의 성적을 기록해, 얼마나 향상 되었는지 학부모와 선생님이 확인할 수 있다.
        (3) 출결 관리
            - 수업이 시작되면 자동으로 출석이 확인되고 학생은 자신의 출석 여부를 확인할 수 있다.
        (4) 수납 관리
            - 수납해야하는 수업료를 확인할 수 있다.
            - `납부하기` 클릭 시, 납부 페이지로 이동한다
        (5) 마이페이지
            - 수업코드를 입력해 수업을 추가할 수 있다.
        (6) 기타
            - 선생님 이름 검색창 옆에 드롭다운 버튼으로 선생님을 선택할 수 있도록 한다.
            - 강의 시작 시간이 되면, 저절로 해당 시간에 진행되는 수업탭으로 전환된다.
    2.3 parents
        (1) 자녀 관리
            - 다자녀인 경우, 자녀별 화면을 확인할 수 있도록 드롭다운 UI를 제공한다.
            - 출결 관리 : 자녀의 출결 현황을 대시보드에서 확인 가능하다.
            - 수업 관리 : 학부모는 성적관리 탭에서 자녀의 시험 성적을 확인할 수 있다.
        (2) 결제
            - 무통장 입금, 카드, 제휴 결제(카카오페이 등) 다양한 결제 수단으로 결제 가능하다.
        (3) 수납 관리
            - 수납해야하는 수업료를 확인할 수 있다.
            - `납부하기` 클릭 시, 납부 페이지로 이동한다.


3. 알림
    3.1 teacher
        - 강사가 원하는 시간대에만 알림을 받을 수 있는 방해금지시간 설정시스템을 구축한다.
        - 학생과 학부모가 강사에게 메시지를 보내면 강사에게 알림이 간다.
        - 학원비 수납 완료시 강사에게 알림메시지가 전송된다
        - 부모님과 학생이 보낸 메시지 알림을 다르게 설정할 수 있다.
    3.2 student
        - 강사가 보낸 메시지에 대한 알림기능을 설정한다.
        - 강사가 클라우드에 자료 업로드시 학생은 알림을 통하여 확인 가능하다.
        - 강사의 알림이 올 시 확인 가능한 소리를 설정한다.
    3.3 parents
        - 자녀의 출석, 수업 종료, 수납일이 다가올 시 학부모에게 알림이 전송된다.
        - 강사가 보내는 채팅 알림을 확인할 수 있게 설정한다.
        - 강사가 보내는 전체 알림메시지를 확인할 수 있는 소리를 설정한다.
        
4. 수납 결제 시스템
    4.1 teacher
        - 월별 정산금 및 수납 내역을 확인할 수 있다.
        - 수납 만료 7일 전을 기준으로, 납부 예정 학생과 납부 완료 학생을 분류한다.
        - 수납 만료 7일 전 학부모에게 정산 알림이 간다.
        - 아임포트 api를 통해 결제한 학생은 납부 완료로 자동 변경한다.
        - 통장으로 직접 송금한 경우에는 강사가 직접 납부완료로 수동 변경한다.
    4.2 student
        - 아임포트 api를 통해 카카오페이, 네이버페이, 토스페이, 통장 입금 등 다양한 결제 수단으로 수업료를 납부한다.
    4.3 parents
        - 아임포트 api를 통해 카카오페이, 네이버페이, 토스페이, 통장 입금 등 다양한 결제 수단으로 수업료를 납부한다.
     
   
5. UI, UX
    5.1 깔끔한 UI의 달력을 사용해, 일정이 한눈에 보일 수 있도록 한다.
        - 활용 예정 API )
        - [https://naver.worksmobile.com/feature/calendar/](https://naver.worksmobile.com/feature/calendar/)
        - [https://nhn.github.io/tui.calendar/latest/tutorial-example03-daily](https://nhn.github.io/tui.calendar/latest/tutorial-example03-daily)
        - [https://fullcalendar.io/](https://fullcalendar.io/)
    5.2 메인화면(로그인 시 가장 먼저 보이는 화면)
        - teacher : 달력 및 일정
        - student : 과제
            - 과제 참여도를 상태바(%)로 표시
            - 사과 아이콘을 클릭해 숙제 제출을 확인하고, 숙제를 제출하면 사과나무에 사과가 달린다.
        - parents : 자녀의 출결 현황, 과제 제출률
    - 모바일에서 편하게 사용할 수 있도록 반응형 웹페이지로 구현한다.
    

6. admin
    - 사이트의 수익창출을 위해 학생수 5명 이상, 그룹과외는 유료로 제공한다.
    - 유료회원 결제는 다양한 결제수단을 제공한다.
    - 유료, 무료회원을 구분하여 조회할 수 있는 화면을 제공한다.

    

7. 채팅
    - teacher
        - 그룹 수업의 경우 학부모, 학생에게 일괄적으로 채팅을 보낼 수 있는 기능 구축한다.
    - student
        - 선생님과의 일대일 채팅으로 수업 내용 질문 가능하다.
    - parents
        - 선생님과 일대일 채팅으로 상담 할 수 있도록 구축한다.
